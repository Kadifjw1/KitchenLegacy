# Кровоток — emissive-стадии 0–5

## Статус модели

**УТВЕРЖДЕНО И ЗАФИКСИРОВАНО.**

Канонические параметры находятся в `MODEL_LOCK.json` и автоматически проверяются скриптом `verify_krovotok_model_lock.py`.

Без отдельного повторного утверждения нельзя изменять:

- 170 элементов модели;
- геометрию, размеры и пропорции;
- `from`, `to`, `origin`, rotation и pivot;
- расположение групп Blockbench;
- диапазон Y `-7.5..31.5`;
- параметры отображения в руках, GUI, рамке и на земле;
- профиль `koloss_approved_display_v1`.

Разрешено изменять только текстуры, emissive-маски, частицы и эффекты, которые не меняют форму меча.

## Что реализовано

- исходная геометрия «Колосса» сохранена без изменений: **170 элементов**;
- программно создаются шесть базовых UV-текстур стадий;
- для каждой стадии создаётся отдельная прозрачная маска свечения;
- центральный кровавый канал дополнен тонкими боковыми прожилками;
- яркость и площадь свечения растут от стадии 0 до стадии 5;
- код рендерит меч двумя проходами: обычная модель + full-bright маска;
- прожилки видны при нулевом игровом освещении;
- существующая механика NBT-зарядов и частиц не изменена.

## Источник

Геометрия берётся из утверждённого `krovotok_JE_1.20.1.bbmodel`, который уже находится в проверенном текстовом архиве задачи Кровотока.

Контрольная сумма геометрии:

`414a5ecdf02b9ded8183e2b02e0c9eb88fa6d9f93e60b15c5573f63e491f2429`

## Материализация

```bash
python3 codex_tasks/krovotok/materialize_krovotok_resources.py
python3 codex_tasks/krovotok/apply_krovotok_display_transform.py
python3 codex_tasks/krovotok/verify_krovotok_model_lock.py
```

Скрипты:

1. восстанавливают и проверяют исходный архив;
2. проверяют 170 элементов модели;
3. создают новый UV-атлас 256×256;
4. создают стадии `stage_0..5`;
5. создают маски `glow_0..5`;
6. создают полный мастер `.bbmodel` в `build/generated/krovotok/krovotok_stages_master.bbmodel`;
7. применяют утверждённое расположение «Колосса»;
8. проверяют модель по зафиксированному каноническому профилю;
9. помещают игровые PNG и JSON в `src/generated/resources`.

Таким образом, Codex и Git diff работают только с текстовым генератором, а бинарные PNG создаются во время подготовки сборки.

## Рендер

`KrovotokItemRenderer` выбирает стадию по `KrovotokCharge`, рендерит базовую модель с обычным `packedLight`, затем ту же геометрию с `LightTexture.FULL_BRIGHT` и прозрачной glow-текстурой.

## Проверка в игре

Проверить стадии 0–5 в GUI, обеих руках, рамке и на земле, отдельно — в полностью тёмном помещении. На стадии 0 прожилки видимы слабо; на стадии 5 основной канал и все ветви должны светиться максимально ярко.
